# Terminal Portfolio Update [MIR_OS]:
- TYPE: Code Cleanup & Mobile Optimizations
- TARGET: Terminal Portfolio
- STATUS: Deployed

# THE_OBJECTIVE
As the project quickly grew, the main files were excessively getting out of hand and finding errors were taking too long. The goal was then to
break up the code into many components and ensure D.R.Y. was being followed.

# THE_EXECUTION
* [CODE_ORGANIZATION]: Took a giant 800-line file and split it up. Now, the virtual file system, the text documents, and the actual terminal commands all live in their own neat, separate files.
* [SMARTER_NAVIGATION]: Built a custom helper function that mimics how a real computer navigates folders. Now, complex commands like `cd ../../logs` work flawlessly.
* [MOBILE_FIXES]: Redesigned the 'quickstart' guide into a compact, two-column dashboard so it doesn't take up the whole screen. Also fixed a bug where long text was stretching the screen horizontally on phones.

# THE_ROADBLOCK
The terminal's logic for reading commands (`cd`, `cat`, and autocomplete) was getting repetitive. Every command had its own slightly different way of figuring out which folder the user wanted to go to, which left room for bugs and made the code messy.

# THE_RESOLUTION
Instead of repeating myself, I extracted that logic into a single 'Path Resolver' tool. Now, all commands share the exact same reliable map to find files. I also set up a cleaner command processor so the main app doesn't have to think about *how* to run a command—it just displays the result.
